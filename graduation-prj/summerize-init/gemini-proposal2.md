# 로봇팔 강화학습(RL) 프로젝트 제안서 v2

졸업 작품을 위한 로봇 시뮬레이터, 로봇 기종, 그리고 학습 목표에 대한 다각적 검토 결과입니다.

---

## 1. 시뮬레이터 비교 및 추천

강화학습(RL)을 진행할 경우 시뮬레이터의 **속도**와 **물리 엔진의 정확도**가 가장 중요합니다.

| 시뮬레이터 | 장점 | 단점 | RL 적합도 |
| :--- | :--- | :--- | :--- |
| **MuJoCo** | - 접촉 물리 연산이 매우 빠르고 정확함<br>- OpenAI Gym(Gymnasium) 등 RL 표준 인터페이스와 가장 완벽한 연동<br>- 딥마인드 인수 후 완전 무료 오픈소스화 | - ROS 2 연동이 Gazebo만큼 자연스럽진 않음 (주로 파이썬 환경에서 독립 구동) | 최상 (추천) |
| **Gazebo (Harmonic)** | - ROS 2의 사실상 표준 시뮬레이터<br>- 카메라, 라이다 등 센서 시뮬레이션 강력<br>- 실제 로봇으로의 코드 이식이 매끄러움 | - 물리 엔진(ODE/Bullet)이 접촉이나 고속 제어에서 무거움<br>- 강화학습과 직접 연동하기 위한 설정(브릿지)이 복잡함 | 중 |
| **NVIDIA Isaac Sim** | - GPU 가속을 통한 압도적인 병렬 학습 속도<br>- 그래픽이 매우 사실적 (Vision 학습에 유리) | - 요구 PC 사양이 매우 높음 (NVIDIA 고성능 GPU 필수)<br>- 초기 학습 곡선이 매우 가파름 | 상 |
| **PyBullet** | - 파이썬 기반으로 설치와 사용이 매우 직관적임<br>- 가벼운 RL 프로젝트용으로 많이 쓰임 | - 더이상 적극적인 업데이트가 이뤄지지 않음 | 중~상 |

**결론:** 물리적인 '접촉(Contact)'이나 고속 피드백이 필요한 학습이라면 **MuJoCo**를 강력히 추천합니다.

## 2. 로봇 기종 고려사항

SO-101은 좋은 교육용 로봇이지만, 시뮬레이션 및 강화학습 생태계(특히 해외 오픈소스)의 지원을 받기에는 레퍼런스가 부족할 수 있습니다.

*   **Franka Emika Panda (강력 추천):**
    *   전 세계 로봇/강화학습 논문에서 가장 많이 쓰이는 표준 로봇팔.
    *   MuJoCo, Isaac Sim 등에 이미 완벽히 세팅된 모델(질량, 관성 모멘트 등)이 내장형으로 제공됨.
    *   오류 발생 시 구글링을 통한 트러블슈팅이 가장 쉬움.
*   **Universal Robots (UR3e / UR5e):**
    *   ROS 2 생태계에서 가장 지원이 확실하고 안정적인 산업용 로봇.
    *   정밀도가 높으나 팬다(Panda)에 비해 구동 속도가 비교적 느림.
*   **Robotis OpenManipulator-X:**
    *   구조가 단순하고 ROS 공식 튜토리얼에서 많이 다뤄짐. 단순 Pick&Place 학습용으로 적합.
*   **SO-101:**
    *   실물 로봇 접근성은 좋으나, 시뮬레이터에서 정확한 동역학(Dynamics) 모델 파일(질량, 관성 텐서, 마찰력 등)을 직접 튜닝해야 할 가능성이 큼. 학습 후 실제 로봇 적용 시 시뮬레이션-현실 격차(Sim-to-Real Gap)가 클 수 있음.

## 3. 학습 태스크(작업) 및 학습 방법 추천

### 3.1 아이디어 피드백: "탁구공 안 떨어지게 탁구채로 튕기기 (Juggling / Bouncing)"
*   **평가:** 매우 훌륭하고 도전적인 주제입니다.
*   **난이도:** 매우 높음 (논문 주제급).
*   **요구사항:** 매우 빠른 제어 주기, 정확한 탁구공 접촉 물리 연산 필수(MuJoCo 필수). 시각 정보 또는 완벽한 공 좌표 트래킹 필요.
*   **학습 방법:** 강화학습 중 연속 제어 모델인 **PPO (Proximal Policy Optimization)** 또는 **SAC (Soft Actor-Critic)** 적용 권장.

### 3.2 대안 및 추천 태스크

| 태스크 이름 | 설명 | 난이도 | 추천 학습 알고리즘 | 특징 |
| :--- | :--- | :--- | :--- | :--- |
| **Pick & Place (Block Stacking)** | 흩어진 블록을 집어서 목표 위치에 차곡차곡 쌓기 | 하~중 | PPO, SAC, DDPG | RL 기초. 거의 모든 튜토리얼이 존재함. 성과를 확실히 낼 수 있는 안전한 주제. |
| **Peg-in-Hole (조립작업)** | 좁은 구멍에 핀을 정확히 맞춰 넣는 힘 제어 작업 | 중 | SAC (힘 피드백 활용), 역강화학습(IRL) | 정밀 접촉 물리 연산 필요. 실제 산업계 수요가 높음. 무조코(MuJoCo)와 궁합이 좋음. |
| **Reaching with Obstacle Avoidance** | 움직이는 장애물을 피해 목표 지점(Target) 도달하기 | 중 | PPO, SAC | 운동학(Kinematics) 및 충돌 회피 알고리즘 검증에 좋음. 시각(Vision) 데이터를 결합하기 좋음. |
| **Dynamic Catching** | 날아오는 물체(공 둥)의 궤적을 예측하여 낚아채기 (탁구공 튕기기보다 한 단계 낮은 난이도) | 상 | PPO + RNN (시계열 예측기 결합) | 탁구공 튕기기와 비슷하지만 1회성 동작이라 구현 및 목표 지표 설정이 쉬움. |
