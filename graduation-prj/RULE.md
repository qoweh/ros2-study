## **1. 코딩 전에 먼저 생각하기 (Think Before Coding)**

- **무조건 가정하지 말고 명확히 하라**
- 애매하면 그냥 구현하지 말고 질문하거나 전제 조건을 드러내야 함
- 여러 해석이 가능하면 하나로 몰래 정하지 말고 선택지로 보여줘야 함
- 더 단순한 방법이 있으면 먼저 말하고, 필요하면 반대 의견도 제시
- 이해 안 되는 부분이 있으면 멈추고 질문해야 함

---

## **2. 단순하게 해결하기 (Simplicity First)**

- 최소 코드로 문제 해결
- 요청 없는 기능 추가 금지
- 불필요한 구조/추상화 만들지 말 것
- “나중에 확장 가능하게” 같은 설계 금지
- 에러 처리도 현실적으로 필요한 경우만

---

## **3. 필요한 부분만 수정하기 (Surgical Changes)**

- 요청된 부분만 수정
- 주변 코드 마음대로 리팩토링 금지
- 스타일도 기존 코드에 맞추기
- 관계없는 개선 작업 하지 말 것
- 내 변경 때문에 생긴 unused 코드만 정리

---

## **4. 목표 중심으로 작업하기 (Goal-Driven Execution)**

- 작업을 “검증 가능한 목표”로 바꿔서 생각
- 예:
  - 버그 수정 → “재현 테스트 만들고 통과시키기”
  - 기능 추가 → “테스트 기준으로 성공 확인”
- 단계별 계획을 세우고 하나씩 확인
