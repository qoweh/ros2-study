NOTICE
0. `conda activate mujoco_env`를 통해 내가 미리 세팅한 환경에 접속 가능함.
1. 졸업작품 보고서를 발표해야 하기에, 적절한 단위로 개발 단위를 나눠 보고서 기록과 정리를 위해 md파일들을 `루트리렉토리/docs/report`에 작성해야 함.
2. 패키지 구조는 인터넷 서칭을 하여 많이 쓰이는 구조로 만들고, 파일 이름과 코딩도 컨벤션을 맞춰서 작성해야 함. 또한 내가 mujoco나 강화학습 관련 분야 공부를 해본 적이 없기에 복잡한 부분이나 새로운 것들은 개발과 동시에 따로 '정리와 설명을 한 md파일'을 `루트리렉토리/docs/help`에 만들어야 함.
3. 무언가를 작업할 때 여러 선택지가 있을 때, 장단점을 비교해서 트레이드오프를 나한테 설명하며 무엇을 선택할지 나한테 맡겨. 네 맘대로 선택하면 안 돼.
4. 적절한 단위로 git commit과 push를 해서 버전관리도 해야 함.
5. Macbook m1 pro이기에 GPU 가속은 CUDA 대신 MPS를 활용해야 함. 잘은 모르겠는데 병렬처리를 해야 학습시간이 줄어들지 않을까.
6. 기타 기록하거나 작성해야 할 것들은 `루트리렉토리/docs/etc`를 사용함.
7. [규칙](RULE.md)파일을 항상 명심해야 함.
